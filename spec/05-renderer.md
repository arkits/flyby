# WebGPU Renderer Specification

Rewrite of `win32ogl/iopengl.c` (OpenGL 1.x backend) for WebGPU.

## Initialization

```typescript
class Renderer {
  device: GPUDevice;
  context: GPUCanvasContext;
  format: GPUTextureFormat; // bgra8unorm
  depthTexture: GPUTexture;
  
  // Pipelines
  litPipeline: GPURenderPipeline;      // Flat-shaded per-face
  unlitPipeline: GPURenderPipeline;    // Flat color, no lighting
  linePipeline: GPURenderPipeline;     // For grid lines, wire smoke
  
  // Uniforms
  viewProjBuffer: GPUBuffer;           // mat4 view-projection
  modelBuffer: GPUBuffer;              // mat4 model transform
  lightBuffer: GPUBuffer;              // vec3 light pos + vec3 camera pos
  
  // Pre-built geometry
  skyGroundBuffer: GPUBuffer;
  gridBuffer: GPUBuffer;
  
  async init(canvas: HTMLCanvasElement): Promise<void>
}
```

### Device Setup

```
1. navigator.gpu.requestAdapter()
2. adapter.requestDevice()
3. canvas.getContext('webgpu')
4. context.configure({ device, format: 'bgra8unorm', alphaMode: 'premultiplied' })
5. Create depth texture (depth24plus, canvas width x height)
6. Create render pipelines
7. Create uniform buffers
8. Build sky/ground and grid geometry
```

## Render Pipelines

### Pipeline 1: Lit (Flat-shaded per-face)

Used for: Aircraft models, field objects (hangars, tower), terrain, PC2 overlays

**Vertex shader input:**
```wgsl
@location(0) position: vec3f,
@location(1) normal: vec3f,    // face normal (flat shading)
@location(2) color: vec3f,     // face color
@location(3) bright: f32,      // 0=lit, 1=unlit
```

**Vertex shader:**
```wgsl
// Transform position by model then view-projection
let worldPos = modelMatrix * vec4f(input.position, 1.0);
output.position = viewProjMatrix * worldPos;
output.worldPos = worldPos.xyz;
output.normal = (modelMatrix * vec4f(input.normal, 0.0)).xyz;
output.color = input.color;
output.bright = input.bright;
```

**Fragment shader:**
```wgsl
if (bright > 0.5) {
  // Unlit: flat color
  return vec4f(color, 1.0);
}
// Lit: ambient + diffuse + specular
let N = normalize(normal);
let L = normalize(lightPos - worldPos);
let V = normalize(cameraPos - worldPos);
let R = reflect(-L, N);

let ambient = 0.3;
let diffuse = max(dot(N, L), 0.0) * 0.6;
let spec = pow(max(dot(R, V), 0.0), 60.0) * 0.3;  // BiPow60: t^60 falloff

let finalColor = color * (ambient + diffuse) + vec3f(spec);
return vec4f(finalColor, 1.0);
```

**Light position** (from FLYBY.C:487-488):
```
light = eye.position + {0, 1000, 0}  // light is always above camera
```

### Pipeline 2: Unlit

Used for: Sky/ground gradient, grid lines, smoke trails, help text

Same vertex shader but fragment just outputs `vec4f(color, 1.0)`.

### Pipeline 3: Line

Used for: Blue grid lines, wire/trail smoke

Primitive topology: `line-list`

## Depth

- Format: `depth24plus`
- Depth test: `less-equal` (matches original `GL_LEQUAL`)
- Depth write: enabled
- Clear value: 1.0

## Draw Calls Per Frame

Following the original `DrawScreen` (FLYBY.C:452-553) order:

```
1. clearScreen()
   - Clear color = sky color from field (or default blue)
   - Clear depth = 1.0

2. drawSkyGround(skyColor, groundColor)
   - Two large triangles forming a horizon quad
   - Uses unlit pipeline
   - Drawn directly in screen/clip space
   - Original splits at horizon line from eye pitch and bank

3. drawNearGroundSupport()
   - Small world-space ground plane near the camera
   - Browser adaptation to keep the airport floor readable
   - Not treated as exact original engine behavior

4. drawFieldOverlays(field, posAtt)
   - Field `PC2` overlays only
   - Uses original overmap pitch adjustment (`-16384`)
   - Drawn before grid and inserted scene objects

5. drawGridLines()
   - 41 lines in X direction, 41 lines in Z direction
   - Range: -20000 to +20000, spacing 1000
   - Color: BiBlue (0,0,1)
   - Uses line pipeline
   - y=0 (ground plane)

6. drawField(field, posAtt)
   - For each SRF: drawSrf(srf, transformedPos)
   - For each terrain: drawTer(ter, transformedPos)
   - For each PLT: drawPc2(pc2, transformedPos)
   - Recursive for nested FLDs

7. drawSrfModel(aircraft, posAtt)
   - Transform model to world space
   - Triangulate each N-gon face into (nVt-2) triangles
   - Send vertex buffer to GPU
   - Use lit pipeline

8. drawSmoke(smokeGeometry)
   - Generated by smoke.ts each frame
   - Uses unlit pipeline
   - Triangles for ribbon/solid, lines for wire/trail

9. drawHelpText()  (if helpCount > 0)
   - 2D overlay text "PRESS X-KEY TO EXIT"
   - Render with canvas 2D overlay or simple textured quad
```

## SRF Model GPU Buffer

For each SrfModel, pre-build a GPU vertex buffer at load time:

```typescript
interface SrfGpuModel {
  vertexBuffer: GPUBuffer;   // Static vertex data
  vertexCount: number;       // Total triangle vertex count
}

function buildSrfGpuBuffer(model: SrfModel): SrfGpuModel {
  // For each polygon:
  //   Triangulate: fan from vertex[0]
  //   For each triangle (v0, vi, vi+1):
  //     Push: position[v0], faceNormal, faceColor, bright
  //     Push: position[vi], faceNormal, faceColor, bright
  //     Push: position[vi+1], faceNormal, faceColor, bright
  
  // Stride: 10 floats per vertex
  // [x, y, z, nx, ny, nz, r, g, b, bright]
}
```

## Matrix Construction

### View Matrix

Built from `eye` PosAtt:

```typescript
function buildViewMatrix(eye: PosAtt): mat4 {
  // 1. Build axis from eye position/attitude
  // 2. View matrix = inverse of eye's model matrix
  //    = rotation(-attitude) * translation(-position)
  // 3. Use rotGtoL for the inverse rotation (heading->pitch->bank order)
}
```

### Projection Matrix

Built from `Projection` struct:

```typescript
function buildProjectionMatrix(prj: Projection, aspect: number): mat4 {
  // Original uses: sx = cx + x*magx/z, sy = cy - y*magy/z
  // This is equivalent to:
  //   fov_x = 2 * atan(0.5 * width / magx)
  //   fov_y = 2 * atan(0.5 * height / magy)
  // With prj.magx = width, prj.magy = height (after 2x in flyby):
  //   fov_x = 2 * atan(0.5) ≈ 53.13°
  //   fov_y = 2 * atan(0.5 * h/w) — varies by aspect
  //
  // Build standard perspective matrix with matching FOV
}
```

### Model Matrix

Built from object's PosAtt:

```typescript
function buildModelMatrix(pos: PosAtt): mat4 {
  // T(pos) * Rz(bank) * Rx(pitch) * Ry(heading)
  // Matches BiRotLtoG rotation order: bank→pitch→heading
}
```

## Rendering a PC2 Overlay

PC2 polygons are 2D (x,z). Render as flat quads on the ground plane (y=0):

```typescript
function drawPc2(pc2: Pc2, pos: PosAtt, renderer: Renderer): void {
  // For each polygon in pc2:
  //   For each pair of triangles (fan from first vertex):
  //     vertex.position = {ver.x, 0.01, ver.y} (slight y offset to avoid z-fighting)
  //     vertex.normal = {0, 1, 0} (pointing up)
  //     vertex.color = polygon.color
  //     vertex.bright = 0 (PC2 overlays are lit normally)
  //   Transform by pos Attitude, then translate by pos Position
}
```
